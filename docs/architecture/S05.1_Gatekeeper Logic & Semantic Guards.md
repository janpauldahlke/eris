
#fcp

1. Objective
    
    Enforce a mathematically rigid contract between Layer 2 (The Orchestrator) and Layer 1 (The HTTP Engine). The Gatekeeper dynamically restricts tool availability based on the active `AgentState`, compiles the allowed tools into Ollama's native API schema, and defensively validates the LLM's output. Crucially, it deploys Semantic Guards to evaluate the logical success of tool executions, preventing infinite loops of technically valid but practically useless API calls.
    
2. Architecture & Design Rules
    

A. The Tool Trait & Registry (`src/tools/mod.rs`)

Every capability is implemented via a standardized Rust trait. We use the `schemars` crate to automatically derive JSON schemas from Rust structs. Dependencies (like DB clients or caches) are injected into the Tool structs during boot, keeping the execution signature clean.

Rust

```
use schemars::JsonSchema;
use async_trait::async_trait;

#[async_trait]
pub trait Tool: Send + Sync {
    /// The rigid NOUN:VERB identifier (e.g., "memory:query")
    fn name(&self) -> &'static str;
    
    /// The description optimized for the LLM's router
    fn description(&self) -> &'static str;
    
    /// Returns the schemars-generated JSON schema
    fn parameters_schema(&self) -> serde_json::Value;
    
    /// The execution payload (Dependencies are held in the struct's internal state)
    async fn execute(&self, args: &str) -> crate::error::Result<String>;
}
```

B. State-Based Routing (The Deterministic Diet)

The Gatekeeper compiles a new list of active tools strictly when the `AgentState` shifts.

- `AgentState::Chat`: `[memory:query, vault:read, vault:write, system:time]`
    
- `AgentState::Reflect`: `[memory:stage, memory:commit, vault:read]`
    
- `AgentState::Idle`: `[memory:commit, vault:read]` _(Note: Raw `vault:write` is physically banned here to prevent autonomous file destruction)._
    
- `AgentState::Recover`: Retains the active toolset of the state that failed, plus an explicit `system:give_up` tool to bail out to the user if the SLM realizes it cannot solve the error.
    

C. Native API Injection (Zero-Prompt Bloat)

Before triggering Layer 1, the Gatekeeper does **not** inject schemas into the text prompt. Instead, it maps the allowed `Tool` trait objects into the JSON array expected by the Ollama API's `"tools"` field.

- _Mechanism:_ The `LlmEngine::generate` trait function (Block S01) accepts the `available_tools_json` array and attaches it directly to the HTTP payload, bypassing the text context window entirely.
    

D. The Validation Firewall & Semantic Guards

When Layer 1 returns its JSON payload (via Ollama's `tool_calls` response field), the Gatekeeper runs a multi-stage validation matrix:

1. **Parse Check:** Is the argument payload valid JSON? _(If no -> `Err(ParseFault)` -> Recovery Loop)_.
    
2. **Authorization Check:** Is the requested tool in the active state list? _(If no -> `Err(SchemaViolation)` -> Recovery Loop)_.
    
3. **Parameter Check:** Do the `args` match the `schemars` definition? _(If no -> `Err(SchemaViolation)` -> Recovery Loop)_.
    
4. **The Semantic Guard (Post-Execution):** In `AgentState::Recover`, if a tool executes successfully but returns a logically empty result (e.g., querying Qdrant for a missing file to fix context, and getting 0 results), the Gatekeeper evaluates this as a semantic failure. It manually increments the `recovery_attempts` counter to prevent the agent from getting stuck in an infinite loop of technically valid but useless queries.
    
5. Acceptance Criteria
    

- [ ] The `schemars` crate successfully derives standard JSON schemas from the Rust argument structs at compile time.
    
- [ ] Shifting from `Chat` to `Reflect` mode successfully alters the `tools` array attached to the Ollama HTTP request, entirely removing unauthorized capabilities (like `vault:write`) from the model's reach.
    
- [ ] If the LLM hallucinates a tool name (e.g., calling `memory:search` instead of `memory:query`), the Gatekeeper instantly traps the execution and yields a `SchemaViolation` error.
    
- [ ] If the LLM provides valid JSON but omits a required argument defined by `schemars`, the Gatekeeper traps the execution and yields a precise `SchemaViolation`.
    
- [ ] **Semantic Guard Test:** Simulating a valid tool call in `Recover` mode that yields an empty logical result successfully increments the `recovery_attempts` integer, eventually triggering the `max_recovery_attempts` bailout.
