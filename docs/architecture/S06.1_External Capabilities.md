
#fcp

1. Objective
    
    Equip the Orchestrator with a high-speed, low-footprint mechanism to retrieve live data from the internet: `web:fetch`. The tool must asynchronously fetch raw HTML, aggressively strip all DOM noise (scripts, styles, navbars), convert the core content into readable Markdown, and strictly limit the token payload to prevent context window exhaustion.
    
2. Architecture & Design Rules
    

A. The AppConfig Amendment (`src/config.rs`)

Before building the tool, we define its physical constraints to prevent hanging threads and blown context windows.

- `web_fetch_timeout_secs: u64` // (e.g., 10) Drop the connection if the site hangs.
    
- `web_fetch_max_bytes: usize` // (e.g., 20480) Roughly 5k tokens. Truncate after this limit.
    

B. The Network Engine (`reqwest`)

We use `reqwest` for the HTTP client. To optimize performance and connection pooling, the `reqwest::Client` is instantiated once at system boot and injected into the Tool struct.

- **Spoofing:** The client must be built with a standard browser User-Agent string to bypass basic bot-protection blocks.
    
- **Timeout Enforcement:** The client must be built with `.timeout(Duration::from_secs(config.web_fetch_timeout_secs))` to guarantee deterministic execution time.
    

C. The Parser (HTML to Markdown)

Raw HTML is a token-burning nightmare. We must parse it down to semantic text before the LLM sees it.

1. The tool receives the `String` of HTML from `reqwest`.
    
2. It pipes the string through a Markdown converter (using the `htmd` or `scraper` crate).
    
3. **The Truncation Guillotine:** It checks the byte length of the resulting Markdown. If it exceeds `config.web_fetch_max_bytes`, it hard-slices the string and appends: `\n\n[SYSTEM WARNING: CONTENT TRUNCATED DUE TO LENGTH LIMITS]`.
    

D. The Argument Struct (Schema Generation)

The JSON schema is automatically derived via the `schemars` crate.

- **State Restriction:** This tool is strictly gated to `AgentState::Chat`. It is physically withheld from the Ollama API payload during `Reflect` or `Idle` modes.
    

Rust

```
#[derive(Deserialize, JsonSchema)]
pub struct WebFetchArgs {
    /// The fully qualified URL to fetch (must include https:// or http://)
    pub url: String,
}
```

E. Error Handling Integration (Agentic vs. System Faults)

We must strictly distinguish between a Rust execution failure and a valid HTTP environment error.

- **Environmental Errors (404, 403, Timeouts):** If the network call completes but yields an HTTP error code (or hits the 10-second timeout), the Rust execution DOES NOT return an `Err()`. It returns the exact error string in the `Ok()` payload (e.g., `Ok("HTTP Error 403: Access Denied. Try a different source.".into())`). This allows the LLM to read the failure in the `[CHAT_STACK]` and decide its next move without triggering the Gatekeeper's hard recovery loop.
    
- **Cognitive Faults (Malformed URLs):** If the LLM passes a string that fails Rust's URL parsing entirely (e.g., `{"url": "not-a-link"}`), the tool returns `Err(FcpError::SchemaViolation)`. The Orchestrator intercepts this and triggers the forced JSON correction loop (Block S04.2).
    

3. Acceptance Criteria

- [ ] The tool successfully fetches a standard article URL, strips all `<script>` and `<style>` tags, and returns clean Markdown to the Orchestrator.
- [ ] Attempting to fetch a massive page successfully triggers the byte-limit truncation, appending the system warning without crashing.
- [ ] Attempting to fetch a broken URL (e.g., a 404 page) cleanly returns the HTTP error status string directly to the LLM's context window (bypassing the Fuckup Loop).
- [ ] A server that takes 30 seconds to respond is successfully aborted by the `reqwest` timeout, instantly returning a timeout error string to the LLM as a valid turn.
- [ ] Passing a malformed URL string successfully triggers a Rust error, shifting the agent into `AgentState::Recover`.
