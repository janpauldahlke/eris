
#fcp

1. Objective Endow the agent with temporal agency (the ability to delegate tasks to its future self) without risking Markdown formatting collapse or autonomous infinite loops. We achieve this by encapsulating a strict, hidden JSON FIFO queue behind three highly constrained executive tools. The Orchestrator intercepts this queue during the `Idle` heartbeat to drive background compute.
    
2. Architecture & Design Rules
    

A. The Physical Medium & Schema (`vaults/{workspace}/.fcp_agenda.json`) The agenda is strictly a JSON file managed natively by Rust. The LLM never sees the raw JSON.

JSON

```
[
  {
    "id": "a1b2",
    "created_at": 1711533600,
    "description": "Read 90_Drops/api_docs.md and summarize the auth endpoints.",
    "status": "pending"
  }
]
```

_Encapsulation Note: Because the LLM never writes this JSON directly, we can safely add `priority`, `due_date`, or `retry_count` fields in V2 without confusing the model's output schema._

B. The Tool Definitions (Domain: Agenda) These tools are registered in the Gatekeeper (Block S05) to bridge the LLM to the JSON queue.

- **`agenda:push`**
    
    - _State:_ Available in `Chat` and `Reflect`.
        
    - _Args:_ `description` (String - max 200 chars).
        
    - _Logic:_ Rust generates a short alphanumeric ID, attaches the current Unix timestamp, and appends the object to the JSON array.
        
    - _Return:_ `SUCCESS: Task [id] queued for background execution.`
        
- **`agenda:list`**
    
    - _State:_ Available in all states.
        
    - _Args:_ None.
        
    - _Logic:_ Rust reads the JSON file and formats the top 5 pending tasks into a dense, token-efficient text string.
        
    - _Return:_ `1. [a1b2] - Read 90_Drops/api_docs.md ... \n 2. [...]`
        
- **`agenda:complete`**
    
    - _State:_ Available _only_ in `Idle` (Background mode).
        
    - _Args:_ `task_id` (String), `result_summary` (String).
        
    - _Logic:_ Rust removes the task from the JSON queue. It takes the `result_summary` and appends it to today's physical episodic log (`10_Episodic/{date}_Tasks.md`).
        
    - _Return:_ `SUCCESS: Task [id] marked complete and logged.`
        

C. The Heartbeat Hook (Upgrading Block S04.3) We must modify the Orchestrator's `Idle` transition logic to act as the task consumer.

- **The Peek:** When `AppConfig.idle_timeout_secs` triggers, the Orchestrator reads `.fcp_agenda.json` _before_ shifting state.
    
- **The Injection:** * If the queue has items, it pops the oldest task and dynamically compiles the `[ACTIVE_STATE]` prompt: _"You are operating autonomously. Execute this task: {task.description}. When finished, use agenda:complete."_
    
    - If the queue is empty, it falls back to the default Ephemeral memory compaction prompt.
        

3. Acceptance Criteria
    

- [ ] The `agenda:push` tool successfully writes a properly structured JSON object to the hidden `.fcp_agenda.json` file without the LLM needing to format the JSON itself.
- [ ] The Gatekeeper strictly blocks the use of `agenda:complete` while the agent is in `AgentState::Chat`, throwing a `SchemaViolation` if attempted.
- [ ] Triggering the `idle_timeout_secs` while tasks are pending successfully injects the oldest task description directly into the background generation prompt.
- [ ] Executing `agenda:complete` successfully purges the ID from the JSON queue and writes the result string to the visible Obsidian `10_Episodic/` folder.
