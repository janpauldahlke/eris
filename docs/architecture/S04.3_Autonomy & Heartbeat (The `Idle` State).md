
#fcp

1. Objective
    
    Enable the Orchestrator to perform background memory maintenance (cache compaction) when the user is away from the terminal. This system features an asynchronous future-cancellation switch to instantly halt background inference when the user returns, followed by a clean state transition to smoothly restore the primary conversational context.
    
2. Architecture & Design Rules
    

A. The Heartbeat Timer (`src/orchestrator/heartbeat.rs`)

A lightweight `tokio` background task monitors the timestamp of the last user `stdin` event.

- **Configuration:** Controlled by an `idle_timeout_secs: u64` field in `AppConfig` (e.g., 900 for 15 minutes).
    
- **The Trigger:** If `current_time - last_input_time > idle_timeout_secs` AND the current state is `WAIT_FOR_USER`, the Orchestrator automatically shifts to `AgentState::Idle`.
    

B. The Context Swap (Amnesia Prevention)

Before the agent begins background chores, it must safely freeze the user's reality.

1. The Orchestrator clones the active `[CHAT_STACK]` and stores it in RAM as `saved_chat_state`.
    
2. It clears the active stack and injects a highly compressed `[IDLE_STATE]` prompt: _"You are operating autonomously in the background. Execute memory compaction on the ephemeral cache."_
    

C. The Async Guillotine & The Wake Sequence

This is the bridge between the Engine's HTTP generation and the user experience.

- **The Kill Switch:** We use a `tokio::sync::watch` channel (or similar signaling mechanism) named `interrupt_rx`. The execution of `LlmEngine::generate` is wrapped in a `tokio::select!`.
    
- **The Intercept:** If the TUI thread detects a keystroke, it fires the `interrupt_tx`. The `select!` macro instantly resolves, violently dropping the `generate` future. This closes the HTTP socket, forcing the Ollama daemon to instantly halt VRAM execution. The Orchestrator catches this and yields `Err(FcpError::Interrupted)`.
    
- **The Wake Sequence (UX):** When the Orchestrator catches the `Interrupted` error, it streams a stylized transition to the `stderr` telemetry feed:
    
    Plaintext
    
    ```
    [SYSTEM] User interrupt received. Halting background generation...
    [SYSTEM] Dropping context. Restoring conversational reality...
    [SYSTEM] Ready.
    ```
    
- **The Restoration:** The Orchestrator restores `saved_chat_state` to the `[CHAT_STACK]`, shifts the state indicator from `Idle` to `Chat`, and unlocks `stdin`.
    

D. The Autonomous Task Ledger (V1 Scope)

While in `Idle`, the Gatekeeper strictly limits the toolset to prevent destructive autonomous actions.

- **Allowed Task (`moka_compaction`):** The agent is prompted to read all staged ephemeral keys nearing their TTL in the Tier 2 cache, synthesize them, and execute the `memory:commit` tool to permanently burn them to the `20_Semantic` or `10_Episodic` folders in the Vault (Tier 3).
    

3. Acceptance Criteria
    

- [ ] The Orchestrator successfully shifts from `Chat` to `Idle` exactly `AppConfig.idle_timeout_secs` after the last terminal input, updating the TUI status indicator.
    
- [ ] During an active `Idle` generation cycle, pressing any key instantly aborts the Ollama HTTP request via `tokio::select!` cancellation, freeing the GPU.
    
- [ ] Upon interruption, the Orchestrator successfully prints the Wake Sequence to the telemetry feed before seamlessly restoring the exact `[CHAT_STACK]` that existed before the Heartbeat triggered.
    
- [ ] If the agent completes a compaction task without being interrupted, it logs the action to `stderr` and shifts back to a dormant `WAIT_FOR_USER` state.
    
- [ ] The background Heartbeat timer correctly listens to the global `CancellationToken` (Block S00.1) and cleanly drops its execution loop if the binary shuts down.
