
#fcp

1. Objective
    
    Centralize all operational thresholds, network paths, and toggles into a single, strongly-typed Rust struct. This struct is instantiated at boot, representing the absolute static truth of the system environment. It is passed safely through the async state machine as an `Arc<AppConfig>`. No magic numbers are permitted in the orchestration logic.
    
2. Architecture & Design Rules
    

A. The Configuration Schema (Pure Data)

This struct must be purely serializable data. Runtime primitives (like `tokio` channels or `CancellationTokens`) are strictly forbidden here to ensure `serde` compatibility.

Rust

```
// src/config.rs
use std::path::PathBuf;

#[derive(Debug, Clone)]
pub struct AppConfig {
    // -- Core Architecture --
    pub workspace: String,            // Active partition (e.g., "default", "qwen_14b")
    pub vault_root: PathBuf,          // Parent directory of all workspaces (e.g., "./vaults/")
    // Note: The active physical vault is dynamically resolved at runtime as: vault_root.join(workspace)
    pub log_level: String,            // Telemetry verbosity (parsed to tracing::Level later)
    
    // -- Layer 1: The Engine (Ollama HTTP) --
    pub ollama_host: String,          // Daemon URL (e.g., "http://localhost:11434")
    pub model_name: String,           // Base model tag (e.g., "llama3.2")
    pub num_ctx: usize,               // Absolute token context limit (e.g., 32768)
    pub generation_timeout_secs: u64, // Network timeout for the async HTTP client
    pub enable_reasoning_fsm: bool,   // FSM logic to parse and strip <think> tags
    
    // -- Layer 2: The Subconscious (Orchestrator) --
    pub condensation_threshold: f32,  // % of num_ctx triggering Reflect (e.g., 0.75)
    pub condensation_target: usize,   // Target tokens for summary (e.g., 300)
    pub max_tool_rounds: u8,          // Loop protection before WAIT_FOR_USER
    pub max_recovery_attempts: u8,    // Bailout limit for the Fuckup Loop
    
    // -- Layer 3: Memory Subsystem (Moka & Qdrant) --
    pub ephemeral_ttl_secs: u64,      // TTL for moka facts in seconds (e.g., 7200)
    pub qdrant_url: String,           // HTTP/gRPC address for Tier 3
    pub qdrant_collection: String,    // Resolved dynamically: fcp_vault_{workspace}
    pub snapshot_interval_secs: u64,  // Moka-to-Disk interval (e.g., 300)
    pub embed_model_name: String,     // Ollama embedding model (e.g., "nomic-embed-text")
}
```

B. The Hierarchy of Truth

The configuration loader must resolve variables in this strict order of precedence:

1. CLI Arguments (Highest priority - e.g., `fcp --vault /tmp/test`)
    
2. Environment Variables (e.g., `export FCP_VAULT=/path`)
    
3. The `.env` File (Located in the execution directory)
    
4. Hardcoded Defaults (Lowest priority)
    
5. Acceptance Criteria
    

- [ ] The `AppConfig` successfully initializes by merging `.env` files, environment variables, and `clap` CLI arguments using a configuration crate (e.g., `figment` or `dotenvy` + `envconfig`).
- [ ] The `qdrant_collection` string is programmatically generated at initialization by concatenating the `"fcp_vault_"` prefix with the `workspace` string.
- [ ] The `ollama_host` and `generation_timeout_secs` are successfully parsed and ready to configure the HTTP client.
- [ ] All numeric thresholds (TTL, PCT, Rounds) are immutable once the `Arc<AppConfig>` is generated, preventing runtime logic drift.
- [ ] The active vault path is successfully resolved at runtime by joining `vault_root` and `workspace`.
