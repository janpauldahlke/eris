
#fcp

1. Objective
    
    Establish the primary behavioral heuristics and linguistic constraints of the agent without using weight-level representation engineering. This block defines the Immutable Anchor in the Obsidian Vault that Layer 2 injects into every Layer 1 prompt assembly. This ensures the baseline persona is mathematically enforced via standard context weighting.
    
2. The Anchor File (`00_Core/Identity.md`)
    
    This file must be created manually or via the first `fcp chat` boot sequence if it does not exist.
    
    Content Schema (Template):
    

- **Role:** [Insert target persona, e.g., "Senior Rust Engineer" or "Submarine Navigator"]
    
- **Tone:** [Insert tonal constraints, e.g., "Clinical and concise" or "Peer-to-peer, zero fluff"]
    
- **Logic:** [Insert reasoning framework, e.g., "Chain of Thought" or "Dialectical Synthesis"]
    
- **Constraint:** [Insert absolute boundaries, e.g., "State 'Data unavailable' if unsure. No hallucinations."]
    
- **Formatting:** [Insert visual output rules, e.g., "High-scannability, use markdown tables, no engagement loops"]
    

3. Architecture & Design Rules
    
    A. The Write Ban (Human-Only Zone)
    
    The agent has zero permissions to edit, append, or overwrite this file. Any attempt by the agent to target `00_Core/Identity.md` (or the `00_Core/` directory generally) via the `vault:write` tool must instantly trigger a hard `FcpError::ToolFault` and route back to the Fuckup Loop. The identity can only be altered by the human user directly in the Obsidian UI.
    

B. The Assembly Injection (`src/orchestrator/prompt.rs`)

During the Context Assembly (Block S05), the Orchestrator reads `00_Core/Identity.md` directly from disk using native `tokio::fs::read_to_string`. (Do not use LLM tool abstractions for this read).

C. The System Concatenation

The Orchestrator must not send _only_ the identity. It must concatenate the identity with the physical engine laws.

The final string mapped to the `system` role in the `LlmEngine::generate` array must be:

`[Contents of Identity.md]` + `\n\n[SYSTEM DIRECTIVE]: You must wrap your internal reasoning in <think>...</think> tags before outputting the final JSON response.`

D. The Boot Fallback

If the file is missing or corrupted, the Orchestrator must return an `FcpError::Config("Identity Anchor Missing in 00_Core/")` and refuse to boot the interactive loop.

4. Acceptance Criteria
    

- [ ] The boot sequence successfully creates `00_Core/Identity.md` with the default structural template if the vault is empty.
    
- [ ] A simulated turn where the agent attempts to execute `vault:write` targeting `00_Core/Identity.md` is successfully trapped and rejected by the Gatekeeper, yielding an error to the model.
    
- [ ] Modifying this file manually as a user successfully alters the agent's tone and scannability behavior in the next `fcp chat` turn without requiring a daemon restart.
    
- [ ] Deleting this file triggers a graceful `FcpError::Config` failure on boot, preventing the agent from running without a defined identity.
    
- [ ] The HTTP payload sent to Ollama correctly maps the concatenated Identity + `<think>` rules to the `system` message role.
